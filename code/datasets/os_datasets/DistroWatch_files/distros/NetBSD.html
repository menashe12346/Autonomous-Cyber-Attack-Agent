NetBSD is a free, secure, and highly portable UNIX-like Open Source operating system available for many platforms, from 64-bit AlphaServers and desktop systems to handheld and embedded devices. Its clean design and advanced features make it excellent in both production and research environments, and it is user-supported with complete source. Many applications are easily available through The NetBSD Packages Collection.

Version: 10.1
Rating: 8
Date: 2025-04-15
Votes: 11


 + Uber-efficient
+ Rock-solid
+ Makes a good server
+ Makes a good minimal desktop
+ Nice CTWM config
+ Quick to install
+ Good ZFS support
+ rc.d is just superior to SysV or that infamous init whose name shall not be said - Simple with simple scripts and basically indestructible
+ pkgsrc rocks, albeit it could offer some interactive way of port configuration like FreeBSD's ports tree
+ Feels more tidy than e.g. FreeBSD
+ Great variety of tools not present in other systems' by default - WoL (wakeonlan), SMART checking (atactl $disk smart status) etc.
+ It still gets better and better

- Wi-Fi support basically non-existent unless you use over 16-year old Wi-Fi cards
- Common Linux DEs don't work as good as they do on Linux - At least MATE and XFCE4
- Lack of guides etc. due to it's very niche. You usually rely exclusively on documentation
- Lack of Unicode in terminal may be irritating (But in X11 it is here)
- It's definitely NOT for beginners, which is normal in case of general-purpose OSes.

It's great under-appreciated toaster-first Unix that definitely worth a try. But remember it's a niche OS which has it's niche-related caveats.

Version: 10.0
Rating: 7
Date: 2024-04-03
Votes: 0


 Good, simple OS that keeps true to original BSD.
However, despite it being able to run on everything, how well it runs depends on the hardware.
It seems to be running really well on ThinkPads, but really seem to struggle a lot on IdeaPads for example.
On the IdeaPads I experience WiFi instabilities, random crashes while on WiFi (not while on ethernet though), segfaults on everything until I reboot, graphical WM's having very random glitches all the time, and more problems.
None of these happened on the ThinkPads however, apart from being unable to connect to WiFi altogether.

The other thing I noticed is that upgrading from 9.3 to 10.0 was very confusing.
The documentation is lacking in their explanation on how to correctly use the `sysupgrade` utility, which as an OpenBSD user is not they way I expect to.
And once you've upgraded, I wasn't sure if the upgrade succeeded or not, because the utility said it did, but after reboot `uname` was still reporting 9.3, and after another reboot it started reporting `10.0`.
